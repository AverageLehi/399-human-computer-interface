01

Slide 1: Absolute vs. Relative Units

Welcome, everyone! Today, we're diving into CSS units. Understanding units is the foundation of building websites that look great whether you are viewing them on a massive desktop monitor or a tiny mobile phone.

In CSS, we divide units into two main categories: absolute (fixed, uncompromising units) and relative (adaptable, contextual units).

02

Slide 2: Pixels (px)

Important: A CSS pixel is not a literal physical glowing dot on your screen. Because retina screens pack lots of physical dots into small spaces, a CSS pixel is actually a standardized measurement, roughly 1/96th of an inch.

Pixels are absolute. If you say font-size: 16px, it stays that size regardless of what parent container it sits inside. It is consistent, but it is not responsive.

Prediction: If we set one text to 16px and another to 32px, what will happen?
  • Will it depend on the parent?
  • Or will it stay fixed?
Practice: Let's give these text boxes some fixed sizes in our IDE. 
.px-text-small { font-size: 16px; }
.px-text-large { font-size: 32px; }

Slide 3: Em (em)

Now we enter the world of relative units. The em unit calculates its math based on the specific font-size of its parent element.

However, you must be incredibly careful. em sizes compound when you nest them. Without CSS applied, the boxes below all look the same size. Let's see what happens when we use em.

Prediction: If the parent is 20px and the child is 2em:
  • What will the child size be?
  • If the grandchild is also 2em, what happens next?
  • Does it stay the same or grow again?
Practice: First, establish the base size of the parent. Then give both the child and grandchild a size of 2em and watch the grandchild compound. 
.em-parent-box { font-size: 20px; }
.em-child { font-size: 2em; }
.em-grandchild { font-size: 2em; }

04

Slide 4: Root Em (rem)

Because the math for em compounding gets too messy, we have an amazing alternative: rem, which stands for root em.

Instead of looking at its direct parent, rem always looks up to the very root of the document, the <html> element. This prevents compounding completely.

Prediction: If we set the root (html) to 16px:
  • What will 2.5rem be?
  • Will it depend on this container?
Practice: Set the root size of the document, then size our text relative to that root. 
html { font-size: 16px; }
.rem-text { font-size: 2.5rem; }
Even if this container was tiny...
This text stays neutral until you add the live CSS.
Prediction Answer

2.5rem becomes 40px because it always uses the root size of 16px. It does not depend on this local container.

Slide 5: Percentages (%)

Percentages define a size relative to the dimensions of a parent container. Right now, our wrapper stretches wide, and the inner box is small.

If we restrict the wrapper, and give the child a percentage, watch how they share a relationship.

Prediction: If the parent is 200px wide:
  • What is 50% of that?
  • What happens if the parent size changes?
Practice: Let's constrain the parent wrapper, and tell the child to only take up 50% of the available width. 
.percent-wrapper { width: 200px; }
.percent-child-box { width: 50%; }

Prediction Answer

If the parent is 200px wide, a child with width: 50% becomes 100px wide. If the parent changes size, the child changes with it.

06

Slide 6: Viewport Units (vw, vh)

Finally, viewport units. The viewport is simply the framing area of your web browser. This unit completely ignores HTML parents and containers.

1vw stands for exactly 1% of the total screen width, and 1vh stands for exactly 1% of the total screen height. Try resizing your browser window.

Prediction: If we use 50vw and 20vh:
  • What happens when we resize the browser?
  • Does it depend on any parent element?
Practice: Let's give our demo box a width of 50vw and a height of 20vh. 
.viewport-demo-box { width: 50vw; height: 20vh; }

Prediction Answer

The box always tracks the browser window. At 50vw it uses half the viewport width, and at 20vh it uses one fifth of the viewport height, regardless of parent containers.

07

Slide 7: Summary and Q&A

Let's recap what we learned:

  • px: A fixed, absolute measurement. Great for exact constraints.
  • em: A unit relative to its parent. Watch out for compounding.
  • rem: A unit relative to the document root. Safe and predictable scaling.
  • %: Fluid layouts based on parent containers.
  • vw/vh: Fluid layouts based directly on the browser window size itself.
