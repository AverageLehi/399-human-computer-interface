Slide 1: Absolute vs. Relative Units

Learn:

CSS units describe how sizes are measured on a page. They mainly fall into two groups.

Type Description Examples
Absolute Stays fixed and does not depend on surrounding elements. px
Relative Changes based on a parent element, the root, or the viewport. em, rem, %, vw, vh

Slide 2: Pixels (px)

Learn:

px is an absolute CSS unit. It gives an element a fixed size that does not change based on its parent container.

Pixels are useful when you want consistent sizing, but they are less flexible than relative units in responsive layouts.

Prediction: If we set one text to 16px and another to 32px, what will happen?
  • Will it depend on the parent?
  • Or will it stay fixed?
Practice: Let's give these text boxes some fixed sizes in our IDE. 
.px-text-small { font-size: 16px; }
.px-text-large { font-size: 32px; }

Slide 3: Em (em)

Learn:

em is a relative unit based on the font-size of its parent element.

When em is used in nested elements, the size can compound, which means each level can become larger again.

Prediction: If the parent is 20px and the child is 2em:
  • What will the child size be?
  • If the grandchild is also 2em, what happens next?
  • Does it stay the same or grow again?
Practice: First, establish the base size of the parent. Then give both the child and grandchild a size of 2em and watch the grandchild compound. 
.em-parent-box { font-size: 20px; }
.em-child { font-size: 2em; }
.em-grandchild { font-size: 2em; }

Prediction Answer

The child becomes 40px because it is 2 times its 20px parent. The grandchild then doubles again from the child, so it becomes 80px. This is the compounding behavior of em.

Slide 4: Root Em (rem)

Learn:

rem means root em. It always measures from the root html size of the page.

Because it does not depend on nearby parent elements, rem is easier to predict than em.

Prediction: If we set the root (html) to 16px:
  • What will 2.5rem be?
  • Will it depend on this container?
Practice: Set the root size of the document, then size our text relative to that root. 
html { font-size: 16px; }
.rem-text { font-size: 2.5rem; }
Demo Area:

This example stays tied to the root size of the page.

This text stays neutral until you add the live CSS.
Prediction Answer

2.5rem becomes 40px because it always uses the root size of 16px. It does not depend on this local container.

Slide 5: Percentages (%)

Learn:

% is a relative unit that measures size based on a parent container.

If the parent changes size, the child element using a percentage changes with it.

Prediction: If the parent is 200px wide:
  • What is 50% of that?
  • What happens if the parent size changes?
Practice: Let's constrain the parent wrapper, and tell the child to only take up 50% of the available width. 
.percent-wrapper { width: 200px; }
.percent-child-box { width: 50%; }

If the parent is 200px wide, a child with width: 50% becomes 100px wide. If the parent changes size, the child changes with it.

Slide 6: Viewport Units (vw, vh)

Learn:

Viewport units are based on the size of the browser window, also called the viewport.

vw follows viewport width, and vh follows viewport height, so these units respond directly to the screen size.

Prediction: If we use 50vw and 20vh:
  • What happens when we resize the browser?
  • Does it depend on any parent element?
Practice: Let's give our demo box a width of 50vw and a height of 20vh. 
.viewport-demo-box { width: 50vw; height: 20vh; }

Prediction Answer

The box always tracks the browser window. At 50vw it uses half the viewport width, and at 20vh it uses one fifth of the viewport height, regardless of parent containers.

Slide 7: Summary and Q&A

Learn:

Each CSS unit has a different sizing reference, so the best choice depends on what should stay fixed and what should adapt.

  • px: A fixed, absolute measurement. Great for exact constraints.
  • em: A unit relative to its parent. Watch out for compounding.
  • rem: A unit relative to the document root. Safe and predictable scaling.
  • %: Fluid layouts based on parent containers.
  • vw/vh: Fluid layouts based directly on the browser window size itself.
Practice: Open the floor to questions. You can freely edit any of the CSS values in the IDE to immediately answer student "what if" scenarios.

Presenter Notes

Teaching Guide

Use this panel as a quick cue sheet while the main page stays clean for presenting.

01 02 03 04 05 06 07

Slide 1

Welcome, everyone! Today, we're diving into CSS units. Understanding units is the foundation of building websites that look great whether you are viewing them on a massive desktop monitor or a tiny mobile phone.

In CSS, we divide units into two main categories: absolute, which stay fixed, and relative, which adapt to context. Use the table on the slide to introduce that difference before moving into specific examples.

Slide 2

Important: a CSS pixel is not a literal physical glowing dot on the screen. On high-density displays, it is a standardized measurement rather than a one-to-one screen dot.

Emphasize that px is absolute. If you set font-size: 16px, it stays that size no matter what parent container it sits inside. It is consistent, but it is not responsive.

Slide 3

Now we enter the world of relative units. The em unit calculates its math based on the specific font-size of its parent element.

Walk through the compounding slowly: if the parent is 20px, then a child at 2em becomes 40px. If the grandchild is also 2em, it doubles again from the child and becomes 80px.

Slide 4

Because the math for em compounding can get messy, this is a good place to introduce rem as the safer alternative.

Explain that rem always looks up to the root html element. It does not read from the local parent, so it avoids compounding completely.

Slide 5

Percentages define a size relative to the dimensions of a parent container, so keep the explanation grounded in that parent-child relationship.

Point out that when the wrapper changes, the child changes with it. That is what makes percentage-based layouts fluid.

Slide 6

Finally, introduce viewport units by contrasting them with percentages. Percentages depend on a parent container, but viewport units ignore parents and follow the browser window itself.

As you explain 1vw and 1vh, resize the browser slightly so students can see the box respond in real time.

Slide 7

Close by recapping the unit references: px is fixed, em reads from the parent, rem reads from the root, percentages read from the parent container, and vw or vh read from the viewport.

Invite quick "what if" edits in the CSS so the recap feels interactive instead of purely verbal.
